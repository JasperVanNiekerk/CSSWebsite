Overflow & Visibility

Learn how to control content overflow and element visibility. These properties are essential for managing content that exceeds its container's boundaries and controlling element visibility without affecting layout.

CSS

.overflow-demo {
  height: 120px;
  width: 200px;
  border: 2px solid var(--demo-border);
  margin: var(--space-2);
  padding: var(--space-2);
  background: var(--demo-bg);
  display: inline-block;
  vertical-align: top;
}

.overflow-demo.visible { overflow: visible; }
.overflow-demo.hidden { overflow: hidden; }
.overflow-demo.scroll { overflow: scroll; }
.overflow-demo.auto { overflow: auto; }

.overflow-x-scroll { 
  overflow-x: scroll; 
  overflow-y: hidden; 
}

.overflow-y-scroll { 
  overflow-x: hidden; 
  overflow-y: scroll; 
}

.visibility-box.hidden {
  visibility: hidden;
}

.overflow-content {
  background: var(--brand-light);
  padding: var(--space-2);
  margin: var(--space-1);
  border-radius: var(--radius-1);
  font-size: var(--fs-sm);
}

Notes

  • overflow: visible (default) - Content extends beyond the container boundaries
  • overflow: hidden - Content is clipped and hidden outside the container
  • overflow: scroll - Scrollbars are always visible, even when not needed
  • overflow: auto - Scrollbars appear only when content overflows
  • visibility: hidden - Element is hidden but still takes up space in the layout
  • overflow-x and overflow-y - Control horizontal and vertical overflow independently
  • Use overflow: auto for most responsive designs to avoid unnecessary scrollbars
