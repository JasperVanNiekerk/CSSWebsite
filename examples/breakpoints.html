Breakpoints

Learn how to define and use consistent breakpoints for responsive design. Breakpoints are the screen sizes where your layout changes to accommodate different devices and screen sizes.

Live Demo

Common Breakpoint Values

These are the most commonly used breakpoint values in responsive design:

Standard Breakpoints

  • Mobile: 0px - 767px (phones)
  • Tablet: 768px - 1023px (tablets)
  • Desktop: 1024px+ (desktops and laptops)
  • Large Desktop: 1200px+ (large screens)

Breakpoint Examples

Different approaches to defining and using breakpoints:

Mobile-First Approach

Start with mobile, enhance upward

This approach starts with mobile styles and adds enhancements for larger screens.

@media (min-width: 768px) { ... }

Desktop-First Approach

Start with desktop, adapt downward

This approach starts with desktop styles and adapts for smaller screens.

@media (max-width: 767px) { ... }

Range-Based Breakpoints

Define specific ranges

This approach defines specific ranges for different device types.

@media (min-width: 768px) and (max-width: 1023px) { ... }

Breakpoint Best Practices

Follow these guidelines for effective breakpoint usage:

Breakpoint Guidelines

  • Use consistent values: Stick to a standard set of breakpoints
  • Mobile-first: Start with mobile and enhance upward
  • Test on devices: Always test on actual devices, not just browser resizing
  • Content-driven: Let content determine breakpoints, not device sizes
  • Use relative units: Prefer em/rem over pixels for breakpoints
  • Document breakpoints: Keep a record of your breakpoint values
  • Consider orientation: Test both portrait and landscape orientations
  • Future-proof: Choose breakpoints that will work with future devices

Code

HTML

<div class="breakpoint-visual">
  <div class="breakpoint-bar mobile-bar">
    Mobile (0px - 767px)
  </div>
  <div class="breakpoint-bar tablet-bar">
    Tablet (768px - 1023px)
  </div>
  <div class="breakpoint-bar desktop-bar">
    Desktop (1024px+)
  </div>
</div>

CSS

/* Mobile-first breakpoints */
.breakpoint-visual {
  display: grid;
  grid-template-columns: 1fr;
  gap: var(--space-2);
  margin: var(--space-4) 0;
}

/* Tablet breakpoint */
@media (min-width: 768px) {
  .breakpoint-visual {
    grid-template-columns: 1fr 1fr;
  }
}

/* Desktop breakpoint */
@media (min-width: 1024px) {
  .breakpoint-visual {
    grid-template-columns: 1fr 1fr 1fr;
  }
}

/* Breakpoint bar styles */
.breakpoint-bar {
  height: 40px;
  display: flex;
  align-items: center;
  justify-content: center;
  border-radius: var(--radius-1);
  font-weight: bold;
  color: white;
}

.mobile-bar {
  background: var(--brand);
}

.tablet-bar {
  background: var(--success);
}

.desktop-bar {
  background: var(--warning);
  color: var(--text);
}

/* Common breakpoint values */

/* Desktop: 1024px+ */
/* Large Desktop: 1200px+ */

Notes

  • Standard breakpoints: 768px (tablet), 1024px (desktop), 1200px (large desktop)
  • Mobile-first: Use min-width media queries, start with mobile styles
  • Content-driven: Let your content determine breakpoints, not device sizes
  • Consistent values: Use the same breakpoints throughout your project
  • Test on devices: Always test on actual devices, not just browser resizing
  • Relative units: Consider using em/rem for breakpoints for better scalability
  • Document breakpoints: Keep a record of your breakpoint values and decisions
  • Future-proof: Choose breakpoints that will work with future devices
