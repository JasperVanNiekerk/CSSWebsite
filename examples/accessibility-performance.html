Performance Tips

CSS performance best practices for responsive design:

CSS Performance

  • Use efficient selectors: .class instead of div.class
  • Avoid deep nesting: Keep selectors shallow
  • Minimize repaints: Use transform and opacity for animations
  • Use CSS custom properties: For better maintainability
  • Optimize images: Use appropriate formats and sizes
  • Minify CSS: Remove unnecessary whitespace and comments
  • Use critical CSS: Load above-the-fold styles first

Code

HTML

<div class="contrast-demo">
  <div class="demo-box good-contrast">
    <h5>Good Contrast</h5>
    <p>This text has sufficient contrast...</p>
  </div>
  <div class="demo-box bad-contrast">
    <h5>Bad Contrast</h5>
    <p>This text has poor contrast...</p>
  </div>
</div>

<div class="reduced-motion-demo">
  <h5>Respects User Preferences</h5>
  <p>This box respects the user's motion preferences...</p>
</div>

CSS

/* Good contrast */
.good-contrast {
  background: var(--bg);
  color: var(--text);
}

/* Bad contrast */
.bad-contrast {
  background: #f0f0f0;
  color: #999;
}

/* Reduced motion support */
.reduced-motion-demo {
  transition: transform 0.3s ease;
}

.reduced-motion-demo:hover {
  transform: translateY(-2px);
}

@media (prefers-reduced-motion: reduce) {
  .reduced-motion-demo {
    transition: none;
  }
  
  .reduced-motion-demo:hover {
    transform: none;
  }
}

Notes

  • Color contrast: Ensure sufficient contrast for readability
  • Reduced motion: Respect user's motion preferences
  • Focus indicators: Provide clear focus states for keyboard navigation
  • Semantic HTML: Use proper HTML elements for better accessibility
  • Alt text: Include descriptive alt text for images
  • Performance: Optimize CSS for faster loading
  • Testing: Test with screen readers and keyboard navigation
