Link States

Learn how to style different link states using CSS pseudo-classes. Proper link styling is essential for user experience and accessibility. Always style all four states: :link, :visited, :hover, and :active.

Live Demo

Individual Link States

Hover over and click these links to see different states in action:

Unvisited Link External Link Visited Link (click me) Active Link (hold down)

:link

Styles unvisited links. This is the default state for links that haven't been clicked yet.

:visited

Styles visited links. Browsers remember which links you've clicked and apply this style.

:hover

Styles links when the mouse hovers over them. Provides visual feedback to users.

:active

Styles links while they're being clicked (mouse button is pressed down).

Navigation Menu Example

A practical example of link states in a navigation menu:

  • Home
  • About
  • Services
  • Contact

Accessibility Note: All links are keyboard accessible and include focus-visible outlines for keyboard navigation.

Code

HTML

<div class="link-demo">
  <a href="#unvisited">Unvisited Link</a>
  <a href="https://example.com" target="_blank">External Link</a>
  <a href="#visited">Visited Link (click me)</a>
  <a href="#active">Active Link (hold down)</a>
</div>

<nav class="navigation-demo">
  <ul>
    <li><a href="#home">Home</a></li>
    <li><a href="#about">About</a></li>
    <li><a href="#services">Services</a></li>
    <li><a href="#contact">Contact</a></li>
  </ul>
</nav>

CSS

.link-demo a {
  padding: var(--space-2) var(--space-4);
  border: 1px solid var(--link-border);
  border-radius: var(--radius-1);
  text-decoration: none;
  transition: all 0.2s ease;
  font-weight: 500;
  min-width: 120px;
  text-align: center;
}

/* Unvisited links */
.link-demo a:link { 
  background: var(--link-bg); 
  color: var(--brand); 
  border-color: var(--brand);
}

/* Visited links */
.link-demo a:visited { 
  background: #f0f0f0; 
  color: #666; 
  border-color: #ccc;
}

/* Hover state */
.link-demo a:hover { 
  background: var(--brand); 
  color: white; 
  border-color: var(--brand);
  transform: translateY(-1px);
  box-shadow: var(--shadow-sm);
}

/* Active state (while clicking) */
.link-demo a:active { 
  background: #004499; 
  color: white; 
  transform: translateY(0);
  box-shadow: none;
}

/* Focus for keyboard navigation */
.link-demo a:focus-visible {
  outline: 2px solid var(--brand);
  outline-offset: 2px;
}

/* Navigation menu styles */
.navigation-demo ul {
  list-style: none;
  display: flex;
  gap: var(--space-2);
  margin: 0;
  padding: 0;
}

.navigation-demo a {
  padding: var(--space-2) var(--space-4);
  text-decoration: none;
  border-radius: var(--radius-1);
  transition: all 0.2s ease;
  font-weight: 500;
}

.navigation-demo a:link {
  color: var(--text);
  background: transparent;
}

.navigation-demo a:visited {
  color: var(--muted);
}

.navigation-demo a:hover {
  background: var(--brand);
  color: white;
}

.navigation-demo a:active {
  background: #004499;
  color: white;
}

Notes

  • Always style all four states for better user experience and accessibility
  • :link - Default state for unvisited links
  • :visited - Applied to links that have been clicked before
  • :hover - Applied when mouse hovers over the link
  • :active - Applied while the link is being clicked
  • Order matters: :link, :visited, :hover, :active (LoVe HAte)
  • Use :focus-visible for keyboard navigation accessibility
  • Include smooth transitions for better user experience
  • Ensure sufficient color contrast for all states
