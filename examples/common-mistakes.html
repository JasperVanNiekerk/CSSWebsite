Common Mistakes

Learn from common responsive design mistakes and how to avoid them. Understanding these pitfalls will help you create better, more robust responsive designs.

Common CSS Mistakes

These are common mistakes to avoid in responsive design:

Common Mistakes to Avoid

  • Fixed widths: Use relative units (%, em, rem) instead of fixed pixels
  • Missing viewport meta: Always include the viewport meta tag
  • Non-responsive images: Use max-width: 100% for images
  • Too many breakpoints: Start with mobile-first, add breakpoints as needed
  • Ignoring touch targets: Make buttons and links touch-friendly (44px minimum)
  • Poor contrast: Ensure sufficient color contrast for readability
  • Not testing on devices: Always test on actual mobile devices
  • Overcomplicating: Keep designs simple and focused

Code

HTML

<!-- Bad: Fixed width -->
<div class="fixed-width">
  This box has a fixed width of 800px...
</div>

<!-- Good: Responsive width -->
<div class="responsive-width">
  This box uses max-width: 100%...
</div>

<!-- Bad: Missing viewport meta -->
<!-- No viewport meta tag -->

<!-- Good: With viewport meta -->
<meta name="viewport" content="width=device-width, initial-scale=1">

CSS

/* Bad: Fixed width */
.fixed-width {
  width: 800px;
  background: var(--brand-light);
  padding: var(--space-2);
  border-radius: var(--radius-1);
  margin: var(--space-2) 0;
}

/* Good: Responsive width */
.responsive-width {
  max-width: 100%;
  background: var(--brand-light);
  padding: var(--space-2);
  border-radius: var(--radius-1);
  margin: var(--space-2) 0;
}

/* Bad: Fixed height */
.bad-height {
  height: 400px;
}

/* Good: Responsive height */
.good-height {
  min-height: 200px;
  height: auto;
}

Notes

  • Avoid fixed widths: Use relative units for responsive design
  • Include viewport meta: Essential for mobile rendering
  • Test on devices: Always test on actual mobile devices
  • Use relative units: %, em, rem instead of fixed pixels
  • Make touch targets large: Minimum 44px for touch interfaces
  • Ensure good contrast: Test color combinations for readability
  • Keep it simple: Don't overcomplicate your designs
  • Progressive enhancement: Start with basic functionality, add enhancements
